Algoritmo MenúPrincipal 
1.	 El procedimiento No recibe parámetros
2.	Inicialización del Menú:
Se define un arreglo de String llamado datosmenuprincipal que contiene las opciones del menú principal.
Inicialización de la Opción:
Se declara una variable opción e inicializa con el valor "0".
3.	Bucle de Menú:
Se inicia un bucle do-while que se repetirá mientras la variable opción no sea igual a "5" (opción de salida).
Despliegue del Menú:
Se llama a la función DesplegarMenu con el título del menú y las opciones, y se almacena la opción elegida en la variable opción.
Verificación de Opción:
Se comprueba si opción es null:
Si es null: Se imprime un mensaje de error indicando que la opción es incorrecta.
Si opción es válida, se utiliza un bloque switch para manejar las diferentes opciones:
Caso "1": Imprime "Menú de Productos".
Caso "2": Imprime "Menú de Punto de Venta".
Caso "3": Imprime "Menú de Inventario".
Caso "4": Imprime "Ventas".
Caso "5": Imprime "Salida del Sistema".
Caso default: Imprime "No existe esta opción".
Condición de Salida:
El bucle se repetirá hasta que la opción elegida sea "5".


Algoritmo MenuProductos  
1.	El procedimiento No recibe parámetros
2.	Inicialización del Menú:
Se crea una lista de opciones para el menú de productos, que incluye "Modificar", "Listado" y "Salida".
Se establece una variable llamada opción, que inicialmente tiene el valor "0".
3.	Bucle de Menú:
Se inicia un bucle que se repetirá mientras la opción elegida no sea "3" (que es la opción de salida).
Dentro de este bucle, se llama a una función que muestra el menú y le pide al usuario que elija una opción.
Verificación de Opción:
Primero, se verifica si el usuario no seleccionó nada (es decir, si la opción es null). En ese caso, se imprime un mensaje de error indicando que la opción es incorrecta.
Si el usuario eligió una opción válida, se utiliza un bloque de decisiones para manejar cada opción:
Opción "1": Si elige "Modificar", se imprime el mensaje "Modificar Precio Producto".
Opción "2": Si elige "Listado", se imprime "Listado Producto".
Opción "3": Si elige "Salida", se imprime "Salida del Sistema".
Opción no válida: Si elige cualquier otra opción, se imprime un mensaje indicando que "No existe esta opción".
Condición de Salida:
El bucle continuará hasta que el usuario elija la opción "3", permitiendo al usuario interactuar con el menú tantas veces como desee antes de salir.


Algoritmo MenuPunto Venta 
1.	El procedimiento recibe un parámetro
String idticket, que representa el número del último ticket.
2.	Inicialización de Variables:
Se definen las variables opcion y membrete, y se inicializa Boolean pago como false.
3.	Generación del Nuevo Ticket:
Se llama a IdTicketSiguiente(idticket) para generar el siguiente número de ticket y se almacena el resultado en idticket.
4.	Se obtiene la fecha actual 
Hacemos uso de la función Fecha() y se almacena en fechadia.
5.	Creación del Membrete:
Se construye el membrete que incluye la fecha y el número de ticket, seguido de una línea separadora.
6.	Se define un arreglo datosmenu 
En esta guardaremos las opciones del menú de punto de venta.
7.	Bucle de Menú:
Se inicia un bucle que se repetirá hasta que la opción elegida sea "5" (salir).
Se llama a DesplegarMenu para mostrar el menú y obtener la opción seleccionada.
Validación de Opción:
Si opcion es null, se imprime un mensaje de error.
Si es válida, se utiliza un bloque switch para manejar las diferentes opciones:
Caso "1": Imprime "Agregar Articulo al Ticket".
Caso "2": Imprime "Eliminar Articulo del Ticket".
Caso "3": Imprime "Listado de Articulos del Ticket".
Caso "4": Imprime "Pagar Ticket y salir", marca pago como true y cambia opcion a "5" para forzar la salida.
Caso "5": Imprime "Salida del Ventas". Si pago es false, muestra un mensaje indicando que no se pagó el ticket y que se eliminará.
Caso default: Imprime "No existe esta opción".
Condición de Salida:
El bucle se repetirá hasta que la opción elegida sea "5".
